Help with Look

Look is a self-hosted photo library app for iPhone and iPad. It connects to a Look server you run on your own private network, usually through Tailscale — your photos never leave your hardware.

Setting up

You need three things: a running Look server, an iPhone or iPad on the same Tailnet or trusted private network, and the server's URL — for example http://machine.tailnet-name.ts.net:5678 or http://100.x.y.z:5678.

  1. Start the Look server on your host machine.
  2. Confirm the server is reachable from another device on your Tailnet.
  3. Open Look on your iPhone or iPad and enter the server URL.
  4. If your server sets API_KEY, enter the same key in Look.
  5. Tap Test connection.

Questions & answers

Look can't connect to my server

Work through this checklist — it resolves nearly every connection problem:

  • Confirm Tailscale is running on both the server machine and your iPhone or iPad.
  • Confirm both devices are signed into the same Tailnet.
  • Open the server URL in Safari on the same device. If Safari can't reach it, the problem is the network, not Look.
  • Confirm the Look server is bound to a reachable interface (HOST=0.0.0.0).
  • Confirm port 5678 is allowed by the host machine's firewall.
  • If your server uses an API key, confirm the key in Look matches the server's API_KEY exactly.
Do I need an API key?

Only if your server sets the API_KEY environment variable. If it does, enter the same value in Look's connection settings. If your server doesn't set one, leave the field empty.

Photos or thumbnails aren't loading

Thumbnails are generated by the server on demand, so a large library takes a little while to warm up on first browse. RAW files are converted to JPEG previews in the background — check the server screen in Look to see conversion progress.

If nothing loads at all, run the connection checklist above.

What file formats does Look support?

JPEG, PNG, HEIC, and RAW formats including ARW (Sony), CR2 (Canon), and NEF (Nikon). RAW files get automatically generated JPEG previews for fast browsing, and you can download either the preview or the original.

Are my photos uploaded to the cloud?

No. Look has no cloud component. The app talks directly to the server you run, over your own network. Photos only leave your server when you explicitly save one to your device's Photos library.

How do favorites, albums, and tags work across devices?

They're stored on your Look server, not on the device — so any device connected to the same server sees the same favorites, albums, and tags.

Where do I get the Look server?

The server is open source: github.com/sladebot/look. The README covers installation, configuration, and how to point it at your photo folders.

How do I report a bug or request a feature?

Use the contact form below, or open an issue on GitHub. Either way, including your device model, iOS version, and what you were doing when the problem happened makes fixes much faster.

Privacy

Look has no accounts and collects no analytics. Your photos stay on your self-hosted server. Messages sent through the form below are used only to reply to you.
